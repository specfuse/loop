# Hashed-denylist core ‚Äî normalize, hash, sliding-window match, loader

**Objective.** Add to `leak_scan.py` the primitives that let a **committed**
hashed denylist replace the gitignored plaintext one on surfaces where the
plaintext is absent (CI, the gate-2 Action): a normalizer, a salted-SHA-256
hasher, a char-sliding-window matcher that preserves the plaintext denylist's
substring-match fidelity, and a loader for the `leak_denylist.hashes` file
format. This WU ships the core only; T02 wires it into `scan_repo` and ships the
generator.

**Context.** This is `FEAT-2026-0024/T01`. Implements the hashing design in
PLAN.md "The hashing design (gate 1 crux)" ‚Äî read that section; it is the
contract. Closes the `#45` CI org-name-coverage gap (LEARNINGS
`[FEAT-2026-0020/G2/leak-guard-surface-asymmetry]`).

The existing plaintext path (`load_denylist`, `_check_patterns`'s
`entry.lower() in line.lower()` substring match at `leak_scan.py:105`) stays
**unchanged** ‚Äî pre-commit runs locally where the plaintext file is present.
This WU adds a parallel hashed path.

The `.hashes` file format (self-describing header + one hash per line):

```
# leak_denylist.hashes ‚Äî committed, generated by `leak_scan.py --hash-denylist`.
# Salted SHA-256 of normalized private-org literals. Obfuscation, not secrecy:
# low-entropy names + a public salt resist rainbow tables only, not a targeted
# brute force by someone who already has the repo. Purpose: catch ACCIDENTAL
# re-introduction. See FEAT-2026-0024 PLAN.md.
# salt: <hex>
# lengths: 10,13
<sha256-hex>
<sha256-hex>
...
```

Normalization (one function, used by BOTH the generator in T02 and the matcher
here so they agree): lowercase, then keep only `[a-z0-9]` characters
(`re.sub(r"[^a-z0-9]+", "", s.lower())`). So `acme-widget-iac` ‚Üí
`acmewidgetiac` (13), `AcmeWidgetApp` ‚Üí `acmewidgetapp` (13),
`acmewidget` ‚Üí `acmewidget` (10), `publishWidget` ‚Üí `publishwidget` (13).

Match algorithm: normalize the scanned line; for each committed length `L` in
the header's `lengths` set, slide an `L`-char window across the normalized line,
hash each window with the header salt, and report a hit if the hash is in the
loaded set. The 10-window over `acmewidgetapp` yields `acmewidget` ‚Äî
preserving the mid-atom substring match that an atom-n-gram approach would miss.

Reference the binding rules under `.specfuse/rules/`. The driver owns git; edit
files only.

**Acceptance criteria.**
1. **Red test (fails on HEAD).** New test file
   `tests/test_hashed_denylist.py::test_known_literal_hashes_and_matches`
   builds a hashed-denylist set from a known literal via the new
   public helpers, then asserts a line **containing that literal embedded in a
   longer string** is flagged. This **fails on HEAD** because the helpers do
   not exist (`ImportError`). Issue #45's "a known literal hashes-and-matches"
   acceptance, made executable.
2. `normalize_token(s: str) -> str` exists and returns the lowercase
   `[a-z0-9]`-only form. Tested: `normalize_token("Acme-Widget_IAC")
   == "acmewidgetiac"`.
3. `hash_token(normalized: str, salt: str) -> str` exists and returns the
   salted SHA-256 hex digest (`hashlib.sha256((salt + normalized).encode())
   .hexdigest()`). Deterministic: same input ‚Üí same digest. Tested.
4. `load_hashed_denylist(path: Path | None = None) -> tuple[str, frozenset[int],
   frozenset[str]]` exists, parses the header (`# salt:`, `# lengths:`) and the
   hash lines, and returns `(salt, lengths, hashes)`. Missing file ‚Üí returns
   `("", frozenset(), frozenset())` (no crash ‚Äî mirrors `load_denylist`'s
   absent-file behavior). Comment/blank lines other than the recognized headers
   are skipped. Tested: missing file; a well-formed file; a file with extra
   comments.
5. `hashed_denylist_hits(line: str, salt: str, lengths, hashes) -> bool` (or a
   helper of equivalent shape) implements the char-sliding-window match. Tested:
   - `test_known_literal_hashes_and_matches` (AC1) ‚Äî embedded literal flagged.
   - `test_mid_atom_substring_matches` ‚Äî a length-10 literal flagged inside a
     length-13 normalized atom (the `acmewidget` ‚äÇ `acmewidgetapp` case).
   - `test_clean_token_does_not_match` ‚Äî a non-denylisted token is NOT flagged
     (issue #45's "a clean token does not"). No false positive.
   - `test_empty_set_no_match` ‚Äî empty `lengths`/`hashes` ‚Üí never matches, no
     crash on any line.
6. A module-level committed default salt constant exists (e.g.
   `_DEFAULT_DENYLIST_SALT`) OR the salt is sourced from the `.hashes` header;
   either way the value used to MATCH is the one read from the header, so a
   regenerated file with a new salt stays self-consistent. Documented inline.
